# Contributing Guidelines

Danke fuer dein Interesse an der Mitwirkung am Patientenverfuegung Skill!

## Arten von Beitraegen

Wir freuen uns ueber:
- **Bug Reports** – Fehlermeldungen mit reproduzierbaren Schritten
- **Feature Requests** – Vorschlaege fuer neue Funktionen
- **Dokumentation** – Verbesserungen der README, SKILL.md oder Uebersetzungen
- **Code-Beitraege** – Pull Requests mit neuen Funktionen oder Bugfixes

## Pull Request Prozess

1. **Fork** das Repository
2. Erstelle einen **Feature-Branch** (`git checkout -b feature/AmazingFeature`)
3. **Commit** deine Aenderungen (`git commit -m 'Add some AmazingFeature'`)
4. **Push** zum Branch (`git push origin feature/AmazingFeature`)
5. Erstelle einen **Pull Request**

## Code-Richtlinien

- Python-Code sollte PEP 8 konform sein
- Templates sollten klar strukturiert und kommentiert sein
- Neue Features sollten im README.md dokumentiert werden
- Teste deine Aenderungen vor dem Commit

## Rechtliche Hinweise

- Dieser Skill bietet **keine Rechtsberatung**. Beitraege sollten diesen Disclaimer nicht unterminieren.
- Aenderungen an den rechtlichen Hinweisen (z.B. § 1827 BGB) sollten sorgfaeltig geprueft werden.
- Persoenliche Daten (z.B. generierte Patientenverfuegungen) sollten nie ins Repository gelangen.

## Fragen?

Erstelle ein Issue oder kontaktiere den Maintainer.

## Code of Conduct

- Sei respektvoll und wertschaetzend
- Konstruktive Kritik ist willkommen
- Fokus auf das Wohl der Nutzer
